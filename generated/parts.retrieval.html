parts.retrieval¶

The retrieval module handles retrieval calculations with ARTS. The retrieval functionality is provided by a class structure that is similar to the one used for Jacobians:

  1. RetrievalCalculation handles the actual calculation of retrievals through the ARTS workspace
  2. RetrievalQuantity defines the general interface for quantities which can be retrieved and how to add them to the list of quantities to retrieve.
  3. RetrievalBase defines an abstract base class for the retrieval classes that encapsulate the settings and code that is specific to the given quantity.

Retrieving quantities¶

So that a quantity can be retrieved, it must inherit from the RetrievalQuantity base class. In this case, it can simply be added to the retrieval attribute of a given simulation:

simulation.retrieval.add(q)

This will add the quantity q to the quantities which will be retrieved. Moreover it will instantiate the retrieval_class class of the quantity q and set the q.retrieval attribute of q.

It is important to note that simulation.retrieval and q.retrieval are of different types: simulation.retrieval is of type RetrievalCalculation and coordinates the interaction with the ARTS workspace required for the retrieval. The attribute q.retrieval of the retrieval quantity holds the settings and retrieval results specific to the quantity q.

Base class for quantity-specific retrieval classes.

The quantity specific retrieval classes are meant to hold settings as well as retrieval results after a calculation.

In order to retrieve any quantity the following must be given:

  1. The a priori mean state xa of the retrieval quantity of the Gaussian distribution representing the a priori assumptions.
  2. A covariance matrix covariance or its inverse, a precision matrix precision, of the Gaussian distribution representing the a priori assumption on the retrieval quantities. It is also possible to specify both, in which case the user has to make sure that they are consistent.

In addition the following attributes can be given to customize the behaviour of the retrieval:

  1. An initial state x0 to start the retrieval iteration.
  2. A lower limit limit_low which is used in each iteration as a lower cutoff for the next iteration step $x_i$.
  3. An upper limit limit_high which is used in each iteration as a lower cutoff for the next iteration step $x_i$.

setup(ws, data_provider, *args, **kwargs)[source]¶

Generic setup method for retrieval quantities.

This method tries to get the a priori vector \(x_a\), the start vector \(x_0\), the covariance and the precision matrix from the data provider and sets them in the given workspace.

If no start vector \(x_0\) is provided, the a priori state is used as initial state for the retrieval iteration.

Of the covariance or precision matrices only one must be specified.

Arguments:

ws(typhon.arts.workspace.Workspace): The ARTS workspace on
which to setup the retrieval.

data_provider: Data provider object to query a priori settings from.

*args and **kwargs are forwarded to the data provider.

add(rq)[source]¶

Add a retrieval quantity to the retrieval calculation.

This registers an atmospheric quantity as a retrieval quantity, which means that parts will try to retrieve the quantity using ARTS’s OEM method instead of querying its value from the data provider.

While the data provider is not required to provide get methods for the retrieval quantity itself, it must provide values for its a priori mean and covariance matrix.

Arguments:

rq(RetrievalQuantity): The retrieval quantity for which to
compute the retrieval.
run(simulation, *args, **kwargs)[source]¶

Setup the retrieval calculation.

This methods performs the setup necessary to run the retrieval calculations on the given workspace. That means it does:

  1. Gather a priori mean states and covariance matrices for all
    registered retrieval quantities.
  2. Define the retrieval inside the given ARTS workspace.
  3. Construct the ARTS inversion iterate agenda.

Arguments:

ws(arts): The typhon.arts.workspace.Workspace
object which to setup for the retrieval.

sensors(list): The sensors to use for the retrieval calculation.

scattering(bool): Whether or not to the forward calculations
involve scattering.
retrieval_provider: The data provider providing the a priori data
for the retrieval.
*args, **kwargs: Arguments and keyword arguments that are passed on
to the data provider.
class parts.retrieval.RetrievalQuantity[source]¶

Bases: parts.jacobian.JacobianQuantity

Abstract interface for quantities for which can be retrieved using ARTS.

Instances of the RetrievalQuantity class can be added to the retrieval quantities of a simulation, which means that parts will try to retrieve their value from a given observation.

The interface for retrieval quantities is implemented by four properties that are shared by all quantities that can be retrieved with ARTS:

  1. transformation: Retrieval quantities can be retrieved not only
    in the units used inside ARTS but also as transformed quantities, e.g. in \(log_{10}\) space. The transformation property hols for each retrieval quantity a parts.jacobian.Transformation object representing the applied transformation.
  2. fixed: Property indicating whether a retrieval quantity should
    be held fixed for a given retrieval run.
  3. retrieval_class: A python class object holding the
    quantity-specific retrieval class that is instantiated and set as the quantities’ retrieval attribute when the quantity is added as retrieval quantity.
  4. retrieval: Instance of the quantities retrieval_class
    holding the retrieval settings and results specific to the quantity and a given retrieval run.
